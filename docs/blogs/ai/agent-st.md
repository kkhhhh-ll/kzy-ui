# 智能体经典架构

现代智能体，核心能力是将大语言模型的推理能力与外部世界联通。它能自主理解用户意图，拆解任务，调用工具，获取信息，执行操作，达到目标。<br>

## ReAct(Reasoning and Acting)

模仿人类解决问题的方式，将推理和行动显示结合，形成一个”思考-行动-观察“的循环。

### 工作流程

”思考-行动-观察“的循环<br>
Thought(思考)：智能体的”内心独白“，它会分析当前情况、分解任务、制定下一步计划，或者反思上一步的结果。<br>
Action(行动)：智能体决定采取的具体动作，通常是调用一个外部工具。<br>
Observation(观察)：执行Action后从外部工具返回的结果。<br>

### 核心循环实现

不断地”格式化提示词->调用LLM->执行动作->整合结果“，直到任务完成或达到最大步数限制。<br>

### 特点

1、高可解释性：透明，通过Thought链，可以清晰地看到智能体每一步的”心路历程“。<br>
2、动态规划与纠错：根据每一步从外部世界获取的Observation动态调整后续的Thought和Action。<br>
3、工具协同能力：天然地将大语言模型的推理能力与外部工具的执行能力结合。

### 缺点

1、对LLM自身能力强依赖<br>
2、执行效率问题，多次调用LLM<br>
3、提示词的脆弱性，整个机制的稳定运行建立在一个精心设计的提示词模板上。<br>
4、可能陷入局部最优<br>

### 使用场景

处理的任务充满不确定性，需要与外部API或网页交互

## Plan-and-Solve

解决思维链在处理多步骤、复制问题时容易”偏离轨道“的问题。<br>

### 工作流程

1、规划阶段：智能体接收到用户问题，分解问题，制定清晰、分步骤的行动计划。<br>
2、执行阶段：严格按照计划中的步骤，逐一执行。<br>

### 使用场景

处理的任务逻辑路径清晰、侧重内部推理和步骤分解

## Reflection

为智能体引入事后的自我校正循环，使其能够像人类一样，审视自己的工作，发现不足，进行迭代优化。<br>

### 工作流程

执行-反思-优化的循环<br>

1、执行：智能体用ReAct或Plan-and-Solve生成一个初步的解决方案，即”初稿“
2、反思：调用一个”评审员“LLM，从多个维度评估”初稿“
3、优化：智能体将”初稿“和”反馈“作为新的上下文，再调用LLM，根据”反馈“修正”初稿“

### 使用场景

处理的任务对最终结果的准确性和可靠性有极高要求
